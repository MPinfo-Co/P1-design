# 業務邏輯分析：[Epic] 公司背景驅動的資安專家功能

## 需求說明
## 功能說明

讓資安專家在分析事件時，會去讀「公司背景」的資料當判斷依據，
並且把「為什麼這樣判斷」顯示給使用者看。

舉例：
- 同樣是「登入失敗」事件，如果是公司內部的服務帳號跑備份失敗，
  資安專家就不會誤報為攻擊，因為公司背景裡有寫「這個帳號是備份用的」
- 事件詳情頁會顯示一段話：「此事件參考規則：XXX（來自知識庫：YYY）」，
  讓使用者知道資安專家是看了哪一條規則做出判斷

## 驗收條件

- [ ] 資安專家分析事件時，會載入它「綁定的知識庫」裡的規則
- [ ] 分析結果會記錄「這個事件對應到哪一條規則、來自哪個知識庫」
- [ ] 事件詳情頁顯示「此事件參考規則」的段落，文字白話、不秀技術細節
- [ ] 至少 3 種代表性事件（內網登入失敗、可疑主機持續活躍、暴力破解）
      在 demo 時都能看到對應規則
- 雛型參考：P1-design/Prototype/ai-detail.html

### 事件詳情頁 ── 「此事件參考規則」區塊

位置：事件詳情 Tab（與「事件摘要」、「建議處置方法」同層），插入於「事件摘要」與「建議處置方法」之間。

**版面配置**

```
┌─────────────────────────────────────┐
│ 事件摘要                             │
├─────────────────────────────────────┤
│ 此事件參考規則                        │  ← 新增區塊
│   • 規則名稱A（來自知識庫：知識庫名稱）  │
│   • 規則名稱B（來自知識庫：知識庫名稱）  │
│   （若無比對到規則則顯示灰字提示）      │
├─────────────────────────────────────┤
│ 建議處置方法                          │
├─────────────────────────────────────┤
│ 關聯日誌摘要                          │
└─────────────────────────────────────┘
```

**顯示規則**

- 每筆已比對到的規則，顯示格式：`規則名稱（來自知識庫：XXX）`
- 若分析時未比對到任何規則，顯示灰色提示文字：`本次分析未比對到知識庫規則`
- 此區塊為唯讀，不提供編輯或互動操作
- 規則內容以白話呈現，不顯示系統 ID 或技術識別碼
