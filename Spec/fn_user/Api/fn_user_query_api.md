# 介面說明

| 項目       | 內容                                       |
| -------- | ---------------------------------------- |
| Method   | GET                                      |
| Endpoint | /api/users                               |

# 傳入參數

## 參數類型：Query Parameter

| 參數           | 型別      | 必填  | 說明                           |
| ------------ | ------- | --- | ---------------------------- |
| **角色職位**     | integer | 否   | 依角色篩選，未傳則回傳全部                |
| **關鍵字**      | string  | 否   | 搜尋 **姓名** / **電子信箱**（ILIKE）  |

# 處理邏輯

## 檢核

| 檢核項目  | 失敗條件          | 回應訊息              |
| ----- | ------------- | ----------------- |
| JWT 驗證 | 未登入或 Token 過期 | 401 未登入或 Token 過期 |
| 權限    | 無 `can_manage_accounts` | 403 您沒有執行此操作的權限 |

## 執行

- 查詢 tb_users，LEFT JOIN tb_user_roles + tb_roles
- 若傳入 **角色職位**，過濾對應角色
- 若傳入 **關鍵字**，過濾 **姓名** ILIKE 或 **電子信箱** ILIKE
- 依 tb_users 建立時間升冪排序

# 傳回結果

| Response | Message  | data                          |
| -------- | -------- | ----------------------------- |
| 200      | 查詢成功     | 名稱, Email, 啟用狀態, 角色[]         |
| 4xx      | 依據檢核邏輯   | 無                             |
