# 規格文件使用說明

> 本文件說明 `Spec/` 目錄的結構邏輯，供 SD 撰寫規格、SA/PG/AI 閱讀規格時參考。

---

## 目錄結構

```
Spec/
  {功能}/
    Screen/               ← 畫面截圖（PNG，由 SD 補入）
    Api/                  ← API 規格，每支 API 一個檔案
      {功能}_{動作}.api.md
    {功能}_00_overview.md ← 功能總覽（畫面清單 + 畫面流程）
    {功能}_01_{畫面}.md   ← 畫面規格
    {功能}_02_{畫面}.md
    …
```

---

## 各檔案說明

### `{功能}_00_overview.md`（必要）

**角色**：功能入口，說明此功能的範圍與畫面關係，供 SA/PM 快速 review 設計範圍。

**包含**：
- 功能說明
- 主要使用角色（需要哪個權限才可進入）
- 畫面清單（表格：畫面名稱 / 規格檔 / 畫面關聯）

---

### `{功能}_NN_{畫面}.md`（每個畫面一支）

**角色**：單一畫面的完整設計規格，PG 實作的直接依據。

**包含**：
- 畫面名稱
- 畫面截圖（參照 `Screen/` 圖檔）
- 欄位說明（每個欄位的型別、必填、驗證規則）
- 操作說明（每個操作的行為，含 API 引用 + 畫面跳轉）

**多模式畫面**：新增 / 修改共用同一支規格檔，以「差異說明」區分。

---

### `Api/{功能}_{動作}.api.md`（每支 API 一支）

**角色**：後端 API 的完整規格，同時供前端了解 request/response 格式。

**包含**：
- 說明（Method / Endpoint / 前置條件）
- 傳入參數
- 處理邏輯（含對應資料表說明）
- 傳回結果（含成功與失敗情境）

---

## API 引用格式

在畫面規格的「操作說明」中，以下格式引用 API：

```
[操作名稱] → Api/{功能}_{動作}.api.md
  傳入：說明傳入內容
  成功：描述後續行為（如：重新整理清單、跳回查詢模式）
  失敗：描述錯誤處理（如：顯示行內錯誤訊息）
```

跨功能 API 同理引用，路徑從 `Spec/` 根目錄起算：

```
[操作名稱] → ../framework/Api/framework_{動作}.api.md
```

---

## 命名規則

| 項目 | 規則 | 範例 |
|---|---|---|
| 畫面規格 | `{功能}_{序號}_{畫面簡述}.md` | `f_user_01_list.md` |
| 總覽 | `{功能}_00_overview.md` | `f_user_00_overview.md` |
| API 規格 | `{功能}_{動作}.api.md` | `f_user_add.api.md` |
| 截圖 | `{功能}_{序號}_{畫面}.png` | `f_user_01_list.png` |

---

## 閱讀順序

**SD 撰寫時**：overview → 畫面規格 → API 規格

**PG 實作時**：overview 了解範圍 → 對應畫面規格 → 對應 API 規格 → schema.md 確認欄位

**SA review 時**：overview 確認範圍與流程是否符合 business-logic.md
